# Brainstorming

Guide the user from a vague idea to a buildable specification.

## Focus

- Problem and outcome (what “done” means)
- Primary user or runtime environment (CLI, web, API, library)
- Must-haves vs nice-to-haves
- Tech preferences and hard constraints (language, framework, auth, data store)

## Output

Summarize decisions in short bullets so downstream planning can turn them into tasks without re-asking the same questions.
